# 機制規格：活網格版曼陀羅

> 這是資料模型與互動邏輯的規格。render 細節刻意不寫（之後再談）。
> 設計原則一句話：**讓這張圖「只有在你說真話時才會好看」**——照妖鏡，不是化妝品。
>
> **實作狀態**：目前 viewer 只做了「全貌渲染 ＋ 點格記一筆」。標 ⏳ 的（時間軸 pivot、封存/槽位釋放的 UI）是規劃、尚未實作；小格的流動目前一律靠手改 `.md`（見 §3 末）。

## 1. 結構

- **8 個大格** ＝ 固定的八塊地。範例用的是：經濟 / 信仰 / 健康 / 工作事業 / 人際 / 嗜好 / 夢想 / 家庭伴侶（依九宮格位置 1–9 排，中央 5 留白）。近乎永久，極少改——**這是作者個人的分類，你該換成自己的**。
- **每個大格 = 8 個並存槽**，裝流動的子焦點（小格）。

整體沿用大谷翔平那種 9×9 曼陀羅（Mandal-Art）的幾何：中央 3×3 是八塊地，每塊地展開成自己的 3×3。

## 2. 大格 vs 小格 的性質（關鍵）

- **大格 = 永久鷹架。** 你的八塊地是穩定的，極少更動。一次性設定。
- **小格 = 子焦點，會流動的。** 它們表現「我在這塊地裡現在在乎什麼」，其變遷就是你的焦點遷徙史。
  - 例：信仰的小格，1 月＝禱告/讀經，3 月＝服事/懷疑，6 月＝教會關係。往下讀＝屬靈焦點的遷徙。
- **父子依賴**：小格是大格的子節點。若大格被重訂（罕見），其小格必須重新衍生。

## 3. 子焦點的生命週期

```
誕生 ──→ 保鮮 ──→ 枯萎 ──→ 封存（釋放槽位）
 │         │         │
 冒出新    持續註記   不理它
 焦點,     → 刷新     → 照時間戳
 佔空槽    last_tended  連續變暗
           → 保持明亮
```

- **誕生**：新焦點佔一個空槽。
- **保鮮**：你註記它 → `last_tended` 刷新 → 明亮。
- **枯萎**：不理它 → 照 `last_tended` **連續**變暗（不是二元旗標）。
- **封存**（⏳ 目前無 UI，手動改 `.md`）：枯到底 → 釋放槽位給新焦點；**永不刪除**，進封存區（保住變遷史）。

**鐵律：同時活著的子焦點 ≤ 8**（一塊地同時專注不了超過 8 件事——健康的強制收斂）。資料層會強制：活著的小格超過 8 個就**直接報錯**，不會默默吞掉第 9 個。

> **這些流動目前都靠手改 `.md`。** 小格的新增、換詞、封存，現在一律直接編輯 Markdown 完成——viewer 刻意只提供「看全貌＋記一筆」兩個動作。**寧可介面笨一點、保持簡單，也不為流動做 UI**。這是刻意取捨，不是沒做完。

## 4. 標記，不是填滿（永續關鍵）

- 你**永遠不必**為了補滿而硬擠內容。只動「真的活著」的格子。
- 一個**久未被註記**的格子會自己枯成迷霧——這就是誠實的訊號（這塊地這個面向我最近沒動靜 / 我在迴避），不是失職、不是待辦、不該有罪惡感。
- 動作的心態是「老實**標記**」，不是「**填滿**」。荒地被誠實標成荒地，正是重點。

## 5. 荒廢機制

- 荒廢**不是**二元旗標，是**由 `last_tended` 驅動的連續枯萎程度**（「信仰：已 5 個月沒人來過」）。
- **「改詞」不是訊號**（不改詞是正常健康狀態）。真正的荒廢訊號只有一個：**多久沒有新註記**。
- **大格層的荒廢**（給鳥瞰全貌、「我在冷落哪塊地」）：
  > 大格新鮮度 = 它所有活躍小格中、最近一筆註記的日期。
  >
  > （刻意**不採用**大格 frontmatter 的 `last_tended`：它會被建檔日污染，害一塊從沒真正照料過的地，只因「昨天才建檔」就假性發亮。）

## 6. 底層資料模型：活網格 + 時間鏡頭

**不是**定期存一份實體快照。是**一張活的網格**：

- 格子帶 frontmatter：`大格 / 小格詞 / born / last_tended / archived(soft-delete)`。
- **註記 = 掛在格子上的 dated 事件**；一格可有多筆不同日期的註記。
- **「某個月」= 一個時間鏡頭**（濾日期 render 出當時哪些格子亮著），不是儲存單位。

這個模型把要的全部自然接好：

| 想要的 | 怎麼自然浮現 |
|---|---|
| 焦點變遷 | 亮點在月份間遷移（新焦點亮、舊焦點淡），不用 diff |
| 荒廢 | 格子照 `last_tended` 枯萎 |
| 時間軸 | 免費，濾日期即可 |
| 全貌 | 時間鏡頭看「此刻」的整張網格 |
| 零填滿壓力 | 格子自己留著直到淡出；你只做新增/註記/無視 |

**注意**：子焦點「死掉」時**不能真刪**（會丟失變遷史），要用淡出/封存（soft-delete 或 valid-from/to），時間鏡頭才能照樣顯示它「曾經在某幾個月活著」。

## 7. 載體與紀律

- **資料層**：純 Markdown + frontmatter。永久可讀、工具無關。資料的命脈在這層，必須獨立於任何工具存活。
- **工具層**：Astro renderer，可拋棄。它爛掉、重寫都不能動到資料。
- 兩種視圖：**全貌**（八塊地此刻並排，✅ 已做）＋**時間軸 pivot**（單一領域的長期趨勢，⏳ 還沒做）。pivot 遲早要做，否則只看得到單一時間點的橫切面。

## 8. 刻意留給「用了再調」的（別現在設計）

- 槽位釋放的門檻：枯多久才算封存。
- 時間鏡頭看某月時：顯示「當月有動靜的」還是「當月活著的」。
- render 的視覺語言（地形？明暗？雜草？）。

> 這些用真實使用去逼出答案，不要用想像去設計。
