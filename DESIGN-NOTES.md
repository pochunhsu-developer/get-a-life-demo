# 設計筆記：做一個「照妖鏡」型的個人工具

這是我替自己做人生盤點工具時，沿路收斂出來的幾個取捨。工具本身很小、很個人，
但底下這幾個決定我覺得能通用，抽出來記在這——給任何想做 PKM / 習慣追蹤 / 自我盤點工具的人參考。

（更私密的動機與我自己的人生內容不在這個 repo；這裡只留可遷移的模式。）

## 1. 先決定專案該多大，再寫 code

- **v1 硬封範圍**：這個工具只做兩件事——「看得到全貌」＋「點一下記一筆」。其餘一律留到「用了真的不夠」再說。
- **紀律**：先用純文字手動跑一個月，確認你真的會回來看，**再**投資寫 renderer。
- 那股「再雕一下資料結構／再加個視覺效果」的衝動，多半是用「**做工具的成就感**」逃避「**工具本來要逼你面對的事**」。給這個現象起個名字（我私下叫它「A 吃 B」），看到就喊停。它是最容易無聲失效的一環。

## 2. 資料用 Markdown + frontmatter，不用 JSON / DB

理由，按重要性：

- **工具無關、永久可讀**：十年後沒有任何 runtime，純文字照樣打得開。
- **人可手改**：不靠 app 也能直接編輯，app 只是加速器不是守門員。
- **git diff 友善**：每次改動天然有版本、有歷史、可回溯。
- **資料與工具徹底分離**：工具可拋棄、可重寫，`rm -rf` 掉整個 viewer 都不該動到資料一根寒毛。

對「給自己長期用」的工具，這條的價值遠大於 DB 帶來的查詢便利。

## 3. 工具是鏡子，不是天平

盤點工具最誘人的歧路，是讓它假裝能幫你「做決策」（排序人生輕重）。但排序需要明確權衡，
而那會殺死探索感、也超出一面鏡子該做的事。**它的職責是照出真相，不是替你權衡。** 想清楚它不做什麼，跟想清楚它做什麼一樣重要。

## 4. carry-forward 會說謊 → 用時間戳驅動衰減

很多追蹤工具讓你「沿用上期」，結果把**荒廢偽裝成穩定**（東西還在格子裡，看起來很健康，其實三個月沒碰）。

解法：新鮮度**不**由「內容有沒有改」判定，而由「**多久沒有新註記**（`last_tended`）」連續衰減。
不改詞是正常健康狀態，不是訊號；唯一的荒廢訊號就是「多久沒有新動靜」。讓圖**只有在你說真話時才會好看**。

## 5. 標記，不是填滿

空格不是失職、不是待辦、不該有罪惡感——它是**誠實的訊號**（這塊這個月我沒動靜／在迴避）。
工具要為「誠實」設計，而不是為「填滿」設計。一旦你開始為了補滿格子而硬擠內容，這面鏡子就失準了。

## 6. 不刪資料，用 soft-delete + 時間鏡頭

一個子項「死掉」時不要真刪（會丟失變遷史），用 `archived` 旗標淡出。
「某個月」不是一個儲存單位，而是一個**濾日期的時間鏡頭**——於是「時間軸 / 焦點變遷史」幾乎免費浮現，不用額外建表、不用 diff。這是一種輕量的 event-sourcing 心態。

## 7. 新習慣會死，寄生的才活得久

週期性儀式（回來記一筆）幾乎沒有多巴胺回饋，最容易默默斷掉。
**別新開一個獨立鬧鐘**——把它寄生在你既有的某個穩定習慣後面（例如某個固定的反省時間）。新習慣的存活率很低，寄生的才撐得久。工具再好，沒有回來的人就是死的。

---

這些是模式，不是教條；它們是為「一個人、長期、誠實面對自己」這個情境調出來的。
你的情境不同，照著想一遍、留下對你成立的就好。
